版本控制也可以很療癒

用最直覺的卡片教學，學會日常必備的 Git 指令。

git add

✨

把變更加入暫存區，準備被提交（commit）。

  • 用途: 選擇要提交的檔案或所有變更
  • 常見場景: 開發完成一個功能後，挑選相關檔案
git add .        # 加入所有變更
git add src/app.js  # 加入指定檔案

git commit

📝

將暫存區的變更建立一次提交，產生版本記錄。

  • 用途: 保存變更並附上有意義的訊息
  • 訊息建議: 使用動詞與目的，如「fix: 修正登入驗證」
git commit -m "feat: 新增首頁輪播"
git commit -am "fix: 修正表單驗證"  # 同時 add 已追蹤檔案

git push

🚀

把本地分支的提交推送到遠端倉庫。

  • 用途: 與團隊同步最新變更
  • 常見場景: 完成一段開發後分享進度
git push origin main
git push -u origin feature/login  # 建立追蹤

git pull

⤵️

從遠端抓取最新變更並合併到本地分支。

  • 用途: 取得最新代碼並快速更新
  • 建議: 先拉取再開新工作，減少衝突
git pull
git pull origin main

git revert

⏪

用一個新的提交來「反轉」指定提交的變更。

  • 用途: 安全回退某次提交，保留歷史
  • 差異: 與 reset 不同，revert 不會改動歷史
git log --oneline           # 找到要反轉的 commit
git revert <commit-hash>    # 生成反轉提交

git checkout

🔀

切換分支或還原檔案到指定狀態。

  • 用途: 在不同分支間工作或還原檔案
  • 補充: 新版 Git 建議用 switch/restore
git checkout main
git checkout -b feature/cart   # 建立並切換新分支
git checkout -- src/app.js     # 還原檔案

git merge

🤝

把另一個分支的提交合併到目前分支。

  • 用途: 整合功能分支到主分支
  • 衝突處理: 先解衝突、測試再提交
git checkout main
git pull
git merge feature/login

小提醒

如果你使用的是較新版本的 Git，分支切換可用 git switch，檔案還原可用 git restore，語意更清楚。
